API response shapes (from Wave 2 routes):
- GET /api/decisions -> { cards: DecisionCardData[], counts: Record<string, number>, pagination: {...} }
- GET /api/decisions/pending-count -> { count: number }
- GET /api/decisions/:id -> DecisionCardData
- POST /api/decisions/:id/approve { modifications? } -> { agentActionId, outcome, audit }
- POST /api/decisions/:id/dismiss { reason } -> { agentMemoryId }
- POST /api/decisions/:id/undo -> { agentActionId, rolledBackAt }
- GET /api/audit/agent-actions -> { rows: AgentAction[], pagination }
- GET /api/audit/agent-actions/:id -> AgentAction (with approver + agentRun joins)
- POST /api/audit/agent-actions/:id/rollback { reason } -> { agentActionId, rolledBackAt }

Tool editableParams convention (read from proposalDraft.toolName):
- draftCourierMessage: ["bodyEnglish"]
- proposeCashReminder: ["bodyEnglish", "amountKd"]
- flagForReview: []
EditDrawer renders an input per editableParams entry pre-filled with proposalDraft.args[paramName].

Backend: PHASE_2_LIVE_TOOLS = { "draftCourierMessage" } -> mirrored in frontend types/decisions.ts as a const set used to render the disabled-Approve tooltip on suspend/penalty cards.
