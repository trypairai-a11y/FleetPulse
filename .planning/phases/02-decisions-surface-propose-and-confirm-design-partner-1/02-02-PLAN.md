<objective>
Wave 2 — Backend API surface for the Decisions inbox + audit log + propose-and-confirm flow.

Builds on Wave 1: monitor agent fills PendingAgentAction; this wave reads them as DecisionCardData and exposes the approve/dismiss/undo lifecycle.

Ships:
1. **`services/decisions/cardProjector.ts`** — `projectPendingAction(pa) -> DecisionCardData`. Maps toolName to DecisionTag (draftCourierMessage->warn, flagForReview->review, proposeCashReminder->cash). Reads driverId from subjectId, loads driver name via tenant-scoped query. Computes `toolIsLive` (only `draftCourierMessage` is true in Phase 2). Includes audit-row preview JSON.
2. **`services/decisions/evidenceCollector.ts`** — `collectEvidence(pa) -> Evidence[]`. Per anomaly class, returns linked entity references: shifts (for late-clockin), violations (for rejection clusters), cash records (for cash mismatch), GPS samples (for GPS-stale).
3. **`routes/decisions.ts`** — Express Router with:
   - `GET /` — paginated cards + counts
   - `GET /pending-count` — sidebar badge
   - `GET /:id` — permalink card
   - `POST /:id/approve` — re-invoke live tool with ctx.userId; write AgentAction; resolve PendingAgentAction
   - `POST /:id/dismiss` — write AgentMemory key=`dismissed:*`; resolve PendingAgentAction
   - `POST /:id/undo` — within 5min, mark AgentAction.outcome=rolled_back; revert PendingAgentAction.resolvedAt
4. **`routes/audit.ts`** — Express Router with:
   - `GET /agent-actions` — paginated audit rows with filters
   - `GET /agent-actions/:id` — full row + joined approver
   - `POST /agent-actions/:id/rollback` — tool-specific rollback handler (limited to OWNER/ADMIN)
5. **`server.ts`** — mount both routers under `/api/decisions` and `/api/audit`.

All routes pass through authMiddleware + tenantScope + rbac (ADMIN/OPS_MANAGER/SUPERVISOR for write; +ACCOUNTANT/VIEWER read-only on audit).

Output: 5 files. Wave 0's `cardProjector.test.ts`, `approveFlow.test.ts`, `dismissFlow.test.ts` turn from RED to GREEN.
