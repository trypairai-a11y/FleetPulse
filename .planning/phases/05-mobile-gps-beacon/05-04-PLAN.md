## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Founder's Cloudflare account → R2 IAM token | Token grants object read+write on the bucket; if leaked, an attacker can read all courier photos |
| Founder's Vercel account → production env vars | Env vars are visible to anyone with Vercel admin access; rotation procedure in scope for post-Phase-5 |
| EAS build → Apple/Google distribution | Founder credentials gate the binary; if EAS account is compromised, malicious builds could ship |

## STRIDE Threat Register (Wave 4 scope)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04-01 | Information disclosure | R2 IAM token leakage via Vercel env var compromise | mitigate | Vercel env vars are project-scoped (not org-shared); rotation procedure: revoke R2 token + issue new one + `vercel env rm/add` then `vercel --prod --yes` |
| T-05-04-02 | Denial of service | EAS build queue saturation prevents urgent native fixes | accept | EAS builds take 10-30 min; for an urgent fix the founder waits the queue. No paid-priority needed at design-partner stage. |
| T-05-04-03 | Tampering | Adversary uploads malicious binary to TestFlight or Play Internal | accept | Apple TestFlight + Google Play Internal both require store review for external invitees; founder-only distribution at this stage means no broader exposure |

This is intentional — Phase 8/11 audit log is the product. |
